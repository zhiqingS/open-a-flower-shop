# 开个花店

《开个花店》是一款移动端优先的轻松种花与花束创作小游戏。

玩家在室外花圃中种植和照料花朵，收获花材后，通过辅助式 DIY 制作花束并交付顾客订单，获得代币后继续购买种子、扩展花材与创作空间。

## 当前目标

在一周、约 10 至 16 小时投入内，完成一个可公开试玩的垂直切片：

```text
接取开业第一单
→ 种植 4 种花材
→ 浇水、施肥、除虫并收割
→ 制作首束花
→ 交付订单并获得代币
→ 使用代币开始下一轮种植
```

第一阶段重点验证：

1. 种花过程是否具有期待感与收获爽感。
2. 辅助式 DIY 是否既有参与感，又能稳定产出漂亮花束。
3. 交付订单后的奖励是否足以驱动玩家再次种植。

## 当前花束制作验证

当前暂停扩展种植、订单和第二轮引导，先把“花束制作”从旧方案中拆出来单独验证。视觉目标参考本地文件：

`视觉验证/GPT-第一轮/风格素材/开个花店素材/Gemini_Generated_Image_9481bn9481bn9481.png`

本轮 `cutout-v02` 的目标非常收敛：

- 使用 Gemini 新返回图作为花束制作验证源图。
- 将花束主体裁成 `template-base`，保留图内白色线稿花朵坑位。
- 从底部素材栏拆出 4 个透明花头 PNG，玩家只拖动这 4 个花头。
- 每个花头可以由系统自动补到 1 至 2 个固定模板位置，验证“玩家投入，系统整理”的模板式插花。
- 不再使用旧版主花素材、旧 AI 资产板、散点槽位、程序包装纸或独立骨架 PNG。

## 平台与技术方向

- 主工程：Cocos Creator 3.8.8 + TypeScript
- 首个公开试玩：移动端优先 Web 构建
- 后续发布目标：微信小游戏、抖音小游戏
- 首版存档：平台本地存档；暂不接入后端、账号、支付或真实广告
- 旧 Phaser 原型：保留在 `prototypes/phaser-bouquet/`，仅作交互决策记录

选择 Cocos 是为了让同一套核心工程可以面向 Web、微信和抖音小游戏构建，同时保留适合 2D 游戏的场景、动画、资源和触屏工作流。Web 仍作为最低成本、最快迭代的首个公开试玩渠道。

```bash
  --args --project /Users/bytedance/Games/First_Mini_Game
```

在 Creator 中打开 `assets/scenes/Main.scene` 并点击预览。纯业务规则测试仍使用 Node.js 24：


```bash

```

```text
http://127.0.0.1:7484/?v=manual
```
