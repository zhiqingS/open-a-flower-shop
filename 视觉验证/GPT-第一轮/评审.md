# 《开个花店》GPT 第一轮视觉验证

## 验证目标

验证 GPT 图像生成是否能够支持以下 MVP 美术需求：

1. 保持同一花朵在不同成长阶段中的视觉一致性。
2. 生成可拆分、可复用的花材与包装组件。
3. 生成具有奖励感和自然美感的完整花束。
4. 验证花朵、状态提示和照料工具在竖屏培育界面中的协调性。

本轮只验证视觉方向与模型能力，不将生成图片直接作为最终游戏资产。

## 视觉规范

- 温暖、柔和的二维手绘游戏插画。
- 花朵简化但可以辨认真实品种。
- 使用平涂色块与轻微纸张、干笔颗粒。
- 使用柔和彩色边缘，不使用黑色硬描边。
- 不采用像素风、照片写实或高光 3D 风格。
- 参考素材只用于提炼视觉语言，不复制具体资产、角色和界面。

## 初步结论

GPT 足以承担 MVP 的概念设计和大部分静态美术生产，但生成结果默认比参考素材更加写实和精细。

这一偏差对完整花束是优势，对培育界面则可能增加资产制作和动画成本。建议使用同一套颜色、光照和纹理规范，但区分两个细节等级：

- 培育界面：轮廓清晰、适度简化，优先保证状态和成长阶段可读性。
- 花束结算与收藏：允许更高细节，用于制造视觉奖励和分享欲望。

竖屏界面验证也表明“显式告知需求、不给玩家留下理解成本”的方向可行：状态气泡、对应工具高亮和中央花朵可以在不使用文字的情况下形成完整引导。

## 下一轮验证

1. 将花材组件生成在纯色背景上，去除背景并验证真实分层组合。
2. 使用固定槽位分别拼出简约、自然松散两种花束构图。
3. 简化竖屏培育主界面的状态图标与工具按钮，降低传统手游 UI 感。
4. 将花束缩放至常见手机显示尺寸，检查满天星等细小元素是否仍然清晰。
5. 使用同一任务测试 Gemini 与 Seedream，比较风格稳定性和返工成本。

## 关键决策

第一轮结果证明视觉路线可行，但不能直接开始批量生产。应先完成独立透明花材和竖屏主界面的第二轮验证，再确定最终细节等级和主力模型。
