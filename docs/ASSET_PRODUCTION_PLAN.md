## 旧路线结论

以下路线已经废弃：

- 从整张花束图或资产板粗裁单枝花材。
- 用完整植物素材拼成花束。
- 用少数 8 个散点槽位代表一整束花。
- 用程序多边形绘制包装纸、丝带或吊牌。
- 先扩玩法，最后再调花束美术。

旧路线的问题不是“还没调好”，而是资产结构不适合做漂亮花束。

## P0 资产范围

第一批只服务花束制作验证页：

| 资产 | 数量 | 用途 |
| --- | ---: | --- |
| 带白色线稿坑位的模板底图 | 1 | 作为花束静态主体 |
| 花头透明 PNG | 4 | 作为玩家拖动对象 |
| 完成预览图 | 1 | 本地检查模板补位后的观感 |
| 透明边缘检查图 | 1 | 检查花头是否带残边 |

## 资产规格

- 画风：清新、明亮、手绘线稿、高精度 2D 插画。
- 色彩：花朵丰富饱满，包装和 UI 克制。
- 背景：游戏内资产必须为透明 PNG。
- 线稿：同一资产板内线稿粗细一致，缩小后仍清楚。
- 光照：统一从上方或左上方柔和照明。
- 锚点：花材以插入点为逻辑锚点；包装以前下方束口为逻辑中心。
- 边缘：透明边缘干净，无底色残留、白边或黑边。
- 用途：每个素材必须是插花用枝材或包装层，不要完整盆栽、完整植物或整束成品图。

## 生产顺序

1. 从 Gemini 新验证图裁出花束主体模板底图。
2. 从底部素材栏拆出 4 个花头透明 PNG。
3. 为 4 个花头配置 1 至 2 个固定模板位置。
4. 输出完成预览图和透明边缘检查图。
5. 在 Cocos 中写单页花束制作验证。
6. 在 `430 x 760` 真实浏览器尺寸拖拽验证。

## 花束模板要求

当前模板配置只包含：

- 资产 id。
- 模板坐标。
- 模板尺寸。
- 主吸附目标。
- 深度和可选多点补位。

首版只有 1 个模板，不提前抽象多模板系统。

## 通过标准

- 运行时只出现新模板和 4 个花头。
- 花头不带黑框、底色块或显眼描边残留。
- 未放入状态使用模板底图中的白色线稿坑位，不出现矩形补丁。
- 拖完 4 个花头后，最终图明显优于旧散点版本。
- 在真实浏览器预览中，第一眼明显优于旧散点版本。

## 风险与应对

| 风险 | 应对 |
| --- | --- |
| AI 单张素材风格不一致 | 先做统一资产板，再拆分 |
| 花材拆分后边缘差 | 每张透明 PNG 单独检查边缘 |
| AI 资产不是像素级原图抠图 | 将其定位为同风格候选资产，商用前人工筛选和修边 |
| 模板限制自由度 | 首版优先漂亮，后续增加包装和模板选择 |
| 包装遮挡不自然 | 包装必须分前后层，不再程序绘制 |
| 花束仍显得稀疏 | 增加系统自动绿叶和填充小花层 |
