# 美术资产生产计划

## 当前目标

下一轮美术工作不追求一次性做完整包，而是先验证首束花的核心假设：

> 四类独立透明花材和包装资产，能否在固定花束槽位中稳定组成一束足够漂亮的花。

只有这个假设成立，后续辅助式 DIY 才值得继续扩展更多花材、包装和花束模板。

## P0 资产范围

第一批只制作新手首单所需资产：

| 类型 | 数量 | 用途 |
| --- | ---: | --- |
| 鲜亮粉色大丽花 | 2 个变体 | 主花，承担视觉焦点 |
| 桃橙、浅黄或香槟粉洋牡丹 | 2 个变体 | 配花，补充暖色体积 |
| 清透蓝色飞燕草 | 2 个枝条变体 | 线条花，形成自然高度和冷色对比 |
| 白色小雏菊 | 2 个小束变体 | 填充花，连接空隙 |
| 通用绿叶 | 2 个变体 | 花束层次和底部遮挡 |
| 奶油包装纸 | 后层 1 个、前层 1 个 | 包住花束并遮挡花茎底部 |
| 浅粉丝带 | 1 个 | 完成感和奖励反馈 |

不在本轮制作：

- 大量花种。
- 完整商城图标。
- 顾客立绘。
- 正式排行榜、活动、充值或广告入口 UI。
- 完全自由 DIY 所需的大量角度和透视资产。

## 资产规格

- 画风：清新、明亮、现代花店感；高精度 2D 插画，不使用像素风。
- 色彩：花朵要色彩斑斓、鲜活、有奖励感；背景、包装和 UI 保持克制以衬托花朵。
- 背景：正式资产必须为透明 PNG；验证图可以保存在 `视觉验证/`。
- 尺寸：单个花材建议不低于 `512 × 512`，线条花可使用竖向画布。
- 锚点：花头类以花茎底部为逻辑锚点；包装前层以底部中心为锚点。
- 边缘：透明边缘干净，无明显底色残留。
- 可读性：缩到手机实际显示尺寸后，主花、配花、线条花和填充花仍能区分。
- 独立性：每个资产应能单独放入 Cocos，不依赖整张花束图。

## 生产顺序

1. 生成首束花资产板，确认整体风格、颜色和复杂度。
2. 生成四类花材的独立透明候选图。
3. 生成包装纸前后层和丝带。
4. 在 Cocos 中建立一个图片版花束拼装验证场景或替换现有程序绘制花朵。
5. 用真实手机比例检查组合效果。
6. 通过后再开始做花圃土地、种植阶段图和更多花种。

## 通过标准

- 首束花第一眼足够漂亮且色彩有吸引力，能成为玩家完成订单的主要奖励。
- 花材组合后轮廓完整，不松散、不像素材拼贴。
- 大丽花或主花有明确焦点，但不压住其他花材。
- 飞燕草能自然形成高度，不像硬插上去的装饰。
- 小雏菊在小屏幕下不变成噪点。
- 包装前层能可靠遮住花茎和底部资产边缘。
- 资产进入 Cocos 后构建 Web、微信、抖音小游戏均无错误。

### 2026-06-13 预览结论

v2 资产板裁图接入 Cocos 后，已经证明“直接从资产板粗裁独立花材”不足以承担正式视觉验证：

- DIY 阶段如果没有包装承托，画面会变成花材素材拼贴，不像玩家正在制作花束。
- 裁出的花材锚点、花茎长度、朝向和比例不统一，放进槽位后容易松散、悬浮或互相打架。
- 包装前层必须在 DIY 阶段就参与遮挡，而不能只在最终交付页出现。
- 当前代码中的压紧槽位、缩小素材和包装遮挡只作为止损原型，不能视为正式美术方案。

下一轮美术资产应改为逐个生成或绘制真正的透明花材，并优先验证“同一套花材在 DIY 画布和交付卡片中都能自然成束”。
