# 美术资产生产计划

## 当前目标

从 v0.4 开始，美术生产围绕 Gemini 概念图路线重设：

`视觉验证/GPT-第一轮/风格素材/开个花店素材/Gemini_Generated_Image_9481bn9481bn9481.png`

当前目标不是批量做很多花种，而是先完成一束真正好看的新手首单花束：

> 通过统一资产板、分层透明资产和模板式插花，让玩家完成的第一束花稳定接近 Gemini 概念图的美观度。

详细设计见：

`docs/BOUQUET_REDESIGN.md`

## 旧路线结论

以下路线已经废弃：

- 从整张花束图或资产板粗裁单枝花材。
- 用完整植物素材拼成花束。
- 用少数 8 个散点槽位代表一整束花。
- 用程序多边形绘制包装纸、丝带或吊牌。
- 先扩玩法，最后再调花束美术。

旧路线的问题不是“还没调好”，而是资产结构不适合做漂亮花束。

## P0 资产范围

第一批只服务新手首单。

### 花材资产

| 资产组 | 最小数量 | 建议尺寸 | 用途 |
| --- | ---: | --- | --- |
| 主花大花头 | 3 | `512 x 512` | 花束视觉焦点 |
| 主花小花头或半遮挡 | 2 | `384 x 384` | 丰富主花层次 |
| 配花花头 | 5 | `384 x 384` | 补充暖色体积 |
| 蓝色线条花枝 | 4 | `512 x 768` | 拉高左右轮廓 |
| 白色小花/雏菊束 | 6 | `384 x 384` | 填缝与精致感 |
| 圆叶/桉叶 | 4 | `384 x 512` | 外侧轮廓和中层遮挡 |
| 蕨叶/尖叶 | 4 | `384 x 512` | 底部与侧边层次 |

### 包装与装饰资产

| 资产组 | 最小数量 | 建议尺寸 | 用途 |
| --- | ---: | --- | --- |
| 后包装纸 | 3 | `768 x 1024` | 花材后方结构 |
| 前包装纸 | 3 | `768 x 768` | 遮挡花茎底部 |
| 蓝白条纹丝带 | 1 | `768 x 384` | 首束花记忆点 |
| 小吊牌 | 1 | `256 x 384` | 花店感和完成感 |
| 完成星光 | 3 | `128 x 128` | 完成瞬间反馈 |

## 资产规格

- 画风：清新、明亮、手绘线稿、高精度 2D 插画。
- 色彩：花朵丰富饱满，包装和 UI 克制。
- 背景：游戏内资产必须为透明 PNG。
- 线稿：同一资产板内线稿粗细一致，缩小后仍清楚。
- 光照：统一从上方或左上方柔和照明。
- 锚点：花材以插入点为逻辑锚点；包装以前下方束口为逻辑中心。
- 边缘：透明边缘干净，无底色残留、白边或黑边。
- 用途：每个素材必须是插花用枝材或包装层，不要完整盆栽、完整植物或整束成品图。

## 生产顺序

1. 生成首束花完整资产板，确认整体风格和复杂度。
2. 人工筛选资产板，标注哪些部分可以拆成游戏资产。
3. 生成或裁切透明 PNG 分层资产。
4. 建立 `bouquet-template-v04` 配置。
5. 在 Cocos 中渲染静态模板，先不接拖拽。
6. 对比 Gemini 概念图，调整资产、位置、层级和包装遮挡。
7. 接入 DIY 拖拽，让玩家逐步填充模板。
8. 交付页复用同一份模板状态。
9. 在 `430 x 760` 和常见窄屏尺寸截图验证。

## 花束模板要求

模板必须配置化，不把内容写死在场景组件里。

建议配置内容：

- 图层 id。
- 资产 id。
- 花艺角色：主花、配花、线条花、填充花、绿叶、包装、装饰。
- 位置。
- 缩放。
- 旋转。
- 深度。
- 是否由玩家投放触发。
- 是否为系统自动层。

首版可以只有 1 个模板，但代码结构要允许后续新增模板。

## 通过标准

- 第一眼明显比旧散点版本好看。
- 花束轮廓接近 Gemini 概念图的饱满扇形。
- 包装纸不是程序感几何块。
- 玩家拖入花材后，花束逐步变丰满，而不是单枝素材堆叠。
- 交付页能明确看出这是玩家刚刚完成的同一束花。
- 在真实浏览器预览中，至少 3 名试玩者认为首束花“愿意继续做下一束”。

## 风险与应对

| 风险 | 应对 |
| --- | --- |
| AI 单张素材风格不一致 | 先做统一资产板，再拆分 |
| 花材拆分后边缘差 | 每张透明 PNG 单独检查边缘 |
| 模板限制自由度 | 首版优先漂亮，后续增加包装和模板选择 |
| 包装遮挡不自然 | 包装必须分前后层，不再程序绘制 |
| 花束仍显得稀疏 | 增加系统自动绿叶和填充小花层 |
