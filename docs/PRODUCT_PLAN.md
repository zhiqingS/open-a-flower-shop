# 《开个花店》产品规划

## 一句话定位

一款面向碎片时间玩家的轻松种花与花束创作游戏：玩家亲手培育花材，通过辅助式 DIY 制作漂亮花束，并交付订单获得继续种植的资源。

## 目标玩家

- 希望利用 1 至 5 分钟碎片时间获得轻松反馈的玩家。
- 喜欢种菜、养成、收集和美观内容的玩家。
- 首批试玩对象为开发者本人、朋友，以及通勤族、宝妈和女学生等潜在目标玩家。

## 核心价值

- 种花提供期待感、日常仪式感和收获爽感。
- 花束 DIY 提供创造感、审美满足和差异化。
- 顾客订单为种植提供明确目标，并将花材转换为可持续资源。
- 游戏没有时间压力、不可逆失败或因离开而产生的惩罚。

## 核心循环

```text
接取订单
→ 获得或购买种子
→ 在花圃种植并进行轻量照料
→ 等待或主动加速成长
→ 收获花材
→ 辅助式 DIY 花束
→ 交付订单
→ 获得代币、种子或包装
→ 开始下一轮种植
```

## 双核心玩法

### 种花

- 完整花圃规模暂定为 `4 × 6` 共 24 块土地。
- 新手阶段只开放 6 块土地，避免首次操作压力。
- 基础操作只包含：种植、浇水、施肥、除虫、收割。
- 花朵成长到关键阶段后暂停，等待玩家亲手见证变化。
- 正式循环中的成长时间可通过观看激励广告直接完成；新手阶段免费加速且不展示广告。
- 玩家不登录时不会损失、掉级或永久枯萎。

### 辅助式 DIY 花束

- 玩家决定投入哪些花材，并通过拖入动作参与花束完成。
- 花材拖入花束后吸附到符合角色的模板槽位。
- 系统根据花束模板自动选择变体、位置、大小、角度、遮挡和包装层，保证成品美观。
- 首版不实现完全自由移动、任意缩放和任意旋转。
- 完成花束后交付订单，自动保存一张花束收藏卡片。

## 新手首单

首个订单暂定名为“开业第一单”，目标花束暂定名为“清晨花束”。

### 花材配方

| 花材 | 花艺角色 | 土地数量 | 目标收获 |
| --- | --- | ---: | ---: |
| 粉色大丽花 | 主花 | 1 | 2 枝 |
| 香槟色洋牡丹 | 配花 | 2 | 4 枝 |
| 蓝色飞燕草 | 线条花 | 1 | 2 枝 |
| 白色小雏菊 | 填充花 | 2 | 4 枝 |

绿叶、包装纸、蓝白条纹丝带和小吊牌由订单自动提供，不要求玩家在新手阶段种植或购买。首版不提供包装自由选择，优先保证第一束花稳定漂亮。

### 新手流程

```text
顾客展示目标花束与订单
→ 系统赠送四种种子并开放六块土地
→ 玩家依次种植四种花材
→ 批量浇水
→ 对指定花材完成一次施肥和一次除虫
→ 免费加速并见证成熟
→ 滑动收割全部花材
→ 将四类花材拖入花束模板，系统吸附并整理
→ 系统补入包装、丝带和吊牌
→ 交付订单，获得代币
→ 使用代币购买下一批种子
```

目标完成时间为 3 至 5 分钟。

## 首单后的第二轮引导

第一轮负责教会玩家完成固定答案；第二轮负责让玩家获得第一次惊喜、第一次自主选择和第一次效率升级。

```text
交付首单
→ 集中领取店铺升级与免费发现奖励
→ 免费发现一款必定未拥有的新花种
→ 使用新旧花种完成第二轮种植与轻量照料
→ 浏览两束足够吸引人的花束灵感，并选择本轮目标
→ 返回花圃时花朵恰好成熟
→ 解锁一键收割并获得足够制作两种花束的库存
→ 快捷制作并交付所选花束
→ 店铺再次升级并解锁新土地
→ 结束强制引导
```

第二轮只新增三个需要玩家理解的概念：

- 免费发现新花种：首次必得未拥有花种，不出现重复。
- 花束目标选择：玩家从两个推荐花束中选择想制作的目标。
- 一键收割：在玩家已经体验逐块收割后提供明确的效率升级与数量爽感。

任务、成就和等级奖励在这一阶段合并为一次集中结算，避免连续弹窗取代种花与花束本身的爽感。

金土地暂不进入永久抽取池。未来优先验证一次性的“阳光祝福”：随机让一块土地在本轮获得金色成熟效果与产量加成，避免永久效率奖励过早破坏经济平衡。

## 经济循环

MVP 只使用一种普通代币。

### 获得代币

- 交付顾客订单。
- 后续可通过成就和广告奖励获得，MVP 不实现。

### 消耗代币

- 购买种子。
- 后续可用于购买肥料、解锁土地和包装。

当前第二轮线框通过首次免费发现提供种子，暂不要求玩家立即消费金币。这样可以先验证奖励连击、发现新花和再次种植是否成立，再决定金币购买种子应放在何处，避免第二轮一次教授过多概念。

花材不能直接使用代币购买，避免玩家跳过种花核心玩法。

## 成长与广告

- 前期成长时间必须明显短于正式循环，优先让玩家看到完整结果。
- 新手第一单所有等待均免费加速。
- 后续激励广告只作为玩家主动选择的加速方式，不应成为每次操作后的强制中断。
- MVP 垂直切片使用“免费加速”模拟广告效果，不接入真实广告 SDK。

## 长期方向

以下方向有价值，但不进入 MVP：

- 逐步解锁完整 24 块土地。
- 不同花朵习性和品质花材。
- 更多花束构图模板、包装和 Combo。
- 花束收藏、橱窗展示与被动收益。
- 每周主题赛、好友展示和轻排行榜。
- 克制的激励广告商业化。

## 竞品差异

《我的花园世界》等竞品证明“种花、收获、花艺、收藏”循环具有市场需求，但《开个花店》不复制其具体表达。

本项目的差异化重点：

- 种植与辅助式 DIY 花束同等重要。
- 花束不是预设配方按钮，而是玩家实际参与制作。
- 界面克制，减少活动、VIP、公会和密集数值入口。
- 强调清新现代的视觉审美、创作与收藏，而不是大型农场扩张和重数值成长。

## 主要风险

| 风险 | 应对 |
| --- | --- |
| 独立花材组合后不够漂亮 | 开发前完成透明花材与槽位拼装验证 |
| 花束模板自由度不足 | 首版先确保成品漂亮，后续再逐步增加包装、色系和模板选择 |
| 4 种花材仍让新手感到繁琐 | 批量浇水、滑动收割，照料事件只发生一次 |
| 种植和 DIY 像两个割裂小游戏 | 使用订单明确连接花材需求、制作与奖励 |
| 与竞品第一眼过于相似 | 避免等距大型农场、密集入口和相似 UI 表达 |
| 一周范围失控 | 严格遵循 `docs/MVP_SPEC.md` 的非目标 |
