# 美术方向与视觉验证

## 当前方向

采用清新、明亮、色彩斑斓的高精度游戏视觉：

- 花束具有接近现实花艺的美观度，但不追求照片级真实。
- 花朵是画面主角，颜色应丰富、鲜活、有明确吸引力，不能整体灰暗或过度奶油化。
- 背景、包装和 UI 可以使用奶油白、浅木色和柔和自然光，用来托住花朵而不是定义花朵色彩。
- 花材颜色清新但饱满，轮廓在手机小屏幕下清晰可辨。
- UI 简洁克制，不使用密集传统手游入口。

## 新手花材

- 粉色大丽花：主花，承担视觉焦点，颜色应鲜亮但不刺眼。
- 香槟色洋牡丹：配花，补充体积与暖色层次，可以偏桃橙、浅黄或香槟粉。
- 蓝色飞燕草：线条花，打破圆形轮廓并增加高度，蓝色应清透明显。
- 白色小雏菊：填充花，连接主花和配花之间的空隙。
- 通用绿叶、奶油包装纸和浅粉丝带：由订单提供。

## 色彩原则

- 花朵优先使用丰富色相：粉、桃、黄、蓝、白、绿共同形成“花店感”。
- 包装、背景和 UI 保持克制，用低饱和暖色衬托花朵。
- 避免整束花只有奶油、米白、浅粉等相近颜色，导致奖励反馈不够兴奋。
- 避免高饱和 UI 抢走花朵注意力；斑斓应该主要发生在花朵本身。
- 在小屏幕下，每种花材应能通过颜色和轮廓被快速区分。

## 美术生产原则

- 参考图和第三方截图仅用于研究，不得发布或直接使用。
- AI 可以用于概念探索和候选资产生成，但最终资产必须经过人工筛选和修正。
- 花材必须使用一致视角、光线、比例和清晰度。
- 独立资产需要透明背景、干净边缘和明确锚点。
- 相同花材至少需要少量朝向或形态变化，避免机械重复。
- 不在首版依赖 AI 实时生成花束。

## 当前最大验证任务

在正式开发大量玩法前，需要证明：

> 四类独立花材能够通过固定槽位组合，稳定形成接近目标图美观度的花束。

## 爽感反馈垂直切片

正式花材资产接入前，先使用可替换的程序表现验证四个关键瞬间的反馈节奏：

| 瞬间 | 最小表现 | 通过标准 |
| --- | --- | --- |
| 发现新花 | 花朵弹性揭晓、聚焦光晕、闪光粒子、短提示音 | 玩家会停下来观察新花，而不是立刻跳过 |
| 花朵成熟 | 六株花按顺序绽放、成熟提示、上扬音阶 | 成熟看起来是一次值得等待的事件 |
| 一键收割 | 花材连续飞入库存、逐块显示 `+数量`、连续提示音 | 玩家能明确感知批量收获与数量增长 |
| 花束完成 | 花材轻微自动整理、完成文案、闪光庆祝、和弦提示音 | 玩家明确感到自己完成了一件作品 |

表现层不得修改库存、奖励或订单规则。动画期间需要短暂锁定重复输入，结束后必须自动恢复操作。

首版提示音使用项目内工具生成的原创短音效，只验证节奏与反馈强度；正式音色后续再替换。

需要重新生成原型音效时，运行 `node tools/generate-sfx.mjs`。

### 所需资产

- 粉色大丽花：至少 2 个变体。
- 香槟色洋牡丹：至少 3 个变体。
- 蓝色飞燕草：至少 2 个枝条变体。
- 白色小雏菊：至少 2 个小束变体。
- 通用绿叶：至少 2 个变体。
- 奶油色包装：后层与前层。
- 浅粉丝带。

### 验证方法

1. 为每类花材生成或绘制独立透明资产。
2. 建立一个固定花束槽位模板。
3. 按新手配方将资产放入槽位。
4. 缩放至真实手机显示尺寸。
5. 检查遮挡、空洞、轮廓、配色和小花辨识度。
6. 与目标花束进行第一眼美感对比。

### 通过标准

- 独立花材不存在明显风格割裂。
- 花束轮廓完整，主花突出，飞燕草形成自然延伸。
- 小雏菊不会在手机尺寸下变成视觉噪点。
- 包装能够遮挡花茎和资产底部。
- 至少 3 名试玩者认为组合花束足够漂亮，并愿意继续制作下一束。

## 禁止事项

- 不复制 `Puni the Florist`、《我的花园世界》或其他游戏的具体资产与 UI。
- 不使用像素风。
- 不让 UI、资源条或活动入口抢夺花朵视觉焦点。
- 不在未验证独立花材组合能力前批量生产大量花种。
